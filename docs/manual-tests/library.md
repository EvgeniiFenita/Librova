# Librova: Браузер Библиотеки — Ручные Сценарии

> Инструкция по запуску и полный реестр сценариев: [ManualUiTestScenarios.md](../ManualUiTestScenarios.md)

## 1. Library Browser Basics

1. Открой `Library`.
Ожидаемое поведение:
- верхняя панель с `Search titles, authors, tags...`, фильтром языка и счетчиком книг остается сверху.
- в левом блоке `Current Library` под путем к библиотеке виден summary `Library: ...`, который показывает общее число managed books и единый total library size в `MB`, включая managed objects и `Database/librova.db`.
- карточки книг отображаются в адаптивной сетке.
- если книг больше начальной порции, при прокрутке вниз grid автоматически догружает следующие книги без отдельной кнопки page navigation.
1.1. Сузь окно до минимально доступной ширины и открой `Book Details`.
Ожидаемое поведение:
- окно не позволяет сузиться до состояния, где `Library` ломается по ширине;
- при открытой панели `Book Details` в grid остаются видимыми как минимум две колонки карточек книг.
1.2. Уменьши окно до минимально доступной высоты.
Ожидаемое поведение:
- окно не позволяет ужаться до состояния, где `Library` теряет вторую строку карточек;
- в основной grid остаются видимыми как минимум две строки карточек книг.
2. Если книги есть, нажми на одну карточку книги.
Ожидаемое поведение:
- справа открывается `Book Details`.
- сетка сжимается, а не перекрывается overlay.
- выбранная карточка после перестройки grid не уезжает из видимой области, а остается примерно по центру текущего viewport.
- если `Book Details` закрыть, сетка снова занимает полную ширину секции `Library`.
3. Нажми на ту же карточку повторно.
Ожидаемое поведение:
- `Book Details` закрывается.
- бывшая выбранная карточка после обратной перестройки grid остается видимой и не уезжает за пределы viewport.
4. Если книг нет, посмотри на основную область.
Ожидаемое поведение:
- показывается пустое состояние `Library is empty` или `Nothing found`, а не пустой сломанный блок.
5. Используй `Search titles, authors, tags...` и кнопку `Filters`.
Ожидаемое поведение:
- сетка обновляется без кнопки `Apply`.
- выдача обновляется прямо во время ввода, без `Enter`.
- после очистки поля поиска полный список книг возвращается сразу.
- счетчик книг показывает общее число результатов текущего запроса или фильтра, а не только число уже загруженных карточек на экране.
- одна строка поиска работает как общий full-text search по `title`, `authors`, `tags` и `description`, а не как отдельный режим только для title/author.
- ввод обычной пунктуации вроде `roadside: ("zone")` не ломает поиск ошибкой и не оставляет browser пустым только из-за FTS syntax parsing.
- кнопка `Filters` в toolbar показывает `Filters`, когда фильтры не выбраны, и `Filters · N` (с золотым/amber цветом) при N активных фильтрах.
- при нажатии на `Filters` открывается popup с разделами `LANGUAGES` и `GENRES`.
- summary `Library: ...` в левом блоке `Current Library` при этом продолжает показывать totals всей текущей библиотеки, а не только текущую page/filter выдачу.
6. Поводи мышью по карточкам и выбери несколько разных книг.
Ожидаемое поведение:
- hover заметен визуально через мягкую rounded-подсветку карточки и аккуратную рамку в форме самой карточки, но не вызывает мерцания, заметного скачка текста или перестройки карточки;
- у уже выбранной карточки hover не добавляет вторую рамку поверх selected-state;
- выбор книги не вызывает массового мерцания всей сетки.

## 2. Full Details Loading

1. В `Library` выбери книгу.
Ожидаемое поведение:
- правая панель сама загружает детали выбранной книги.
- панель остается справа внутри секции `Library`, а не уезжает за экран или в отдельную прокручиваемую область.
- отображаются richer metadata, например publisher, year, language, format, series и genres, если они доступны.
- размер файла отображается в `MB`, а не в raw bytes.
- аннотация показывается в отдельном прокручиваемом блоке.

## 3. Library Sort Controls

1. Открой секцию `Library`, убедись, что в библиотеке есть несколько книг.
Ожидаемое поведение:
- в панели инструментов рядом с фильтром языка виден выпадающий список сортировки с вариантами `Title`, `Author`, `Date Added`.
- правее выпадающего списка виден квадратная кнопка с иконкой стрелки вверх (сортировка по возрастанию).
- по умолчанию выбрано `Title`, книги расположены в алфавитном порядке по названию (А→Я / A→Z).
2. Нажми кнопку направления сортировки (стрелка вверх → стрелка вниз).
Ожидаемое поведение:
- иконка меняется на стрелку вниз (по убыванию);
- сетка немедленно обновляется, книги располагаются в обратном алфавитном порядке;
- scroll сбрасывается в начало.
3. Нажми кнопку направления снова (стрелка вниз → стрелка вверх).
Ожидаемое поведение:
- иконка возвращается к стрелке вверх;
- сетка снова показывает книги по возрастанию.
4. Выбери `Author`, затем нажми кнопку направления несколько раз.
Ожидаемое поведение:
- каждый раз список обновляется и scroll сбрасывается;
- иконка корректно переключается между ▲ и ▼ при каждом нажатии.
5. Выбери `Date Added`, установи направление `по убыванию` (стрелка вниз). Закрой и снова открой приложение.
Ожидаемое поведение:
- после перезапуска восстанавливается и ключ сортировки (`Date Added`), и направление (стрелка вниз / по убыванию).
6. Переключись последовательно на все три варианта сортировки: `Title`, `Author`, `Date Added`.
Ожидаемое поведение:
- каждый раз grid обновляется и scroll возвращается в начало;
- кнопка направления сохраняет своё текущее состояние при смене ключа сортировки.
7. Выбери любую сортировку, отличную от `Title`. Перейди в `Settings`, смени путь к библиотеке (или верни обратно), вернись в `Library`.
Ожидаемое поведение:
- после смены корня библиотеки выбранная сортировка и направление сохраняются.

## 2.1 Covers And Legacy FB2 Metadata

1. Импортируй `FB2`, у которого есть извлеченная cover, сохраненная как object в `Objects/<shard>/<shard>/...cover...`.
Ожидаемое поведение:
- в карточке книги отображается реальная cover, а не placeholder gradient;
- cover сохраняет свои пропорции и не обрезается;
- если вокруг cover остаются поля, они выглядят как нейтральный matte background, а не как цветные gradient bands.
2. Выбери эту книгу в `Library`.
Ожидаемое поведение:
- в `Book Details` тоже отображается реальная cover;
- cover сохраняет свои пропорции и не получает цветных полос placeholder background по краям.
3. Импортируй `FB2` с русскими метаданными и `encoding="windows-1251"`.
Ожидаемое поведение:
- импорт завершается без crash;
- книга появляется в `Library`;
- title и author отображаются корректным Unicode-текстом, без битой кодировки и без ошибки `String is invalid UTF-8`.

## 2.2 Placeholder Covers Stay Fully Visible

1. Импортируй книгу без встроенной cover, у которой название начинается с высокой кириллической буквы, например `Война`.
2. Открой `Library` и найди карточку этой книги в grid.
Ожидаемое поведение:
- placeholder cover показывает первую букву названия;
- буква не обрезается сверху или снизу;
- placeholder остается визуально центрированным внутри cover area.
3. Выбери эту книгу и посмотри на `Book Details`.
Ожидаемое поведение:
- в правой панели placeholder-буква тоже отображается полностью;
- размер placeholder адаптируется к доступной cover area без clipping.

## 2.3 Фасетная панель фильтров (Languages + Genres)

1. Открой `Library`, убедись что в библиотеке есть книги на разных языках и с разными жанрами.
Ожидаемое поведение:
- в toolbar секции `Library` рядом с строкой поиска видна кнопка `Filters`;
- кнопка отображает `Filters` без счётчика, если фильтры не выбраны.
2. Нажми на кнопку `Filters`.
Ожидаемое поведение:
- открывается popup с двумя разделами: `LANGUAGES` (сверху) и `GENRES` (снизу);
- каждый раздел содержит список языков / жанров из текущего result set в виде pill-кнопок (chips) без выбранного состояния (= все значения включены);
- у раздела `GENRES` есть строка поиска `Search genres…` для фильтрации списка;
- в нижней части popup видна строка с числом выбранных фильтров и кнопкой `Clear all`.
3. Выбери один язык, например `en`.
Ожидаемое поведение:
- popup остаётся открытым;
- grid сразу обновляется и показывает только книги на выбранном языке;
- кнопка `Filters` меняет label на `Filters · 1` с amber-цветом;
- другие языки в секции `LANGUAGES` не исчезают из списка.
4. Не закрывая popup, выбери ещё один язык, например `ru`.
Ожидаемое поведение:
- grid обновляется и показывает книги на `en` или `ru` (OR-семантика);
- кнопка отображает `Filters · 2`.
5. Выбери один жанр в секции `GENRES`, например `sci-fi`.
Ожидаемое поведение:
- grid обновляется: показываются только книги на выбранных языках И с жанром `sci-fi`;
- счётчик в кнопке увеличивается до `Filters · 3`;
- другие жанры в секции `GENRES` не исчезают из списка.
6. Введи текст в строку `Search genres…` внутри popup.
Ожидаемое поведение:
- список жанров в секции `GENRES` сразу фильтруется по введённому тексту без закрытия popup;
- при очистке строки поиска полный список жанров восстанавливается;
- уже выбранные жанры остаются отмеченными.
7. Нажми `Clear all` в нижней части popup.
Ожидаемое поведение:
- все chips в обоих разделах снимаются;
- grid возвращает полный result set (без языкового или жанрового фильтра);
- кнопка `Filters` возвращает label `Filters` без счётчика.
8. Открой popup, выбери несколько фильтров, затем кликни за пределами popup.
Ожидаемое поведение:
- popup закрывается (light-dismiss);
- выбранные фильтры остаются активными, кнопка показывает `Filters · N`.
9. Закрой и снова открой приложение при активных фильтрах.
Ожидаемое поведение:
- состояние фильтров сбрасывается (сессия не персистируется — штатное поведение).
10. Введи текст в строку поиска при активных фильтрах языка и жанра.
Ожидаемое поведение:
- текстовый поиск применяется поверх языкового и жанрового фильтра;
- список доступных языков и жанров в popup перестраивается под текущий search result set.
